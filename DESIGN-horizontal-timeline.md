# Horizontal Timeline Layout

> 수평 타임라인 레이아웃 상세 설계.
> Canvas Layout Architecture(DESIGN-canvas-layout.md)의 첫 번째 구체 구현.

---

## 1. 공간 구조

캔버스 공간이 수평 축(axis)을 기준으로 두 영역으로 나뉜다.

**축 (Axis)** — 캔버스 좌표계의 고정된 Y 위치에 수평선이 그려진다. 시간은 왼쪽에서 오른쪽으로 흐른다.

**축 위 (Period Zone)** — 기간(period) 노드가 밴드 형태로 배치되는 영역. 축에 바로 붙어서 위로 쌓인다.

**축 아래 (Occurrence Zone)** — 사건(occurrence) 노드가 카드 형태로 배치되는 영역. 축에서 아래로 일정 간격을 두고 배치된다.

```
         ┌──────────────────────────────────────────────┐
         │                                              │
         │    ┌──────────┐  ┌────────────────────┐      │
         │    │  기간 A   │  │    기간 B           │      │
         │    └──────────┘  └────────────────────┘      │
  axis   ├──●──────────●──────────●───────●─────────────┤
         │  ╎          ╎          ╎       ╎             │
         │  ╎          ╎          ╎       ╎  (connector)│
         │ ┌──┐      ┌──┐      ┌──┐    ┌──┐            │
         │ │사건│      │사건│      │사건│    │사건│            │
         │ │ A │      │ B │      │ C │    │ D │            │
         │ └──┘      └──┘      └──┘    └──┘            │
         │                                              │
         └──────────────────────────────────────────────┘
              ← 시간 흐름 (X축) →
```

## 2. 두 종류의 노드 렌더링

타임라인에서는 노드의 **역할(role)**에 따라 렌더링 방식이 완전히 달라진다. 역할은 캔버스의 layout_config 필드 매핑에서 archetype별로 지정된다.

### Period (기간) 노드

축 위에 반투명 색상의 **밴드(band)**로 그려진다. 일반 NodeCard가 아니다.

- 위치: 축 바로 위. start ~ end 시간 범위가 밴드의 시작점과 너비를 결정한다.
- 크기: 가로 = 시간 범위 × 스케일, 세로 = 고정 높이.
- 라벨: 밴드 중앙에 텍스트로 표시.
- 색상: Concept의 color 또는 Archetype 기본색. 반투명.
- 축 마커: 없음. 밴드 자체가 시각적 표현.
- 인터랙션: 클릭(선택), 더블클릭(에디터 열기), 우클릭(컨텍스트 메뉴), 양쪽 끝 드래그(기간 조절).

### Occurrence (사건) 노드

축 아래에 일반 **NodeCard**로 그려진다. 기존 NodeCardDefault와 동일한 컴포넌트 사용.

- 위치: X = 시간값 × 스케일, Y = 축 아래 기본 오프셋. 사용자가 Y를 조절할 수 있음(레인 이동).
- 축 연결: 노드 상단에서 축까지 dashed 수직 커넥터 선.
- 축 마커: 축 위에 해당 시간 위치에 작은 원(●) 표시.
- 인터랙션: 기존 NodeCard와 동일 + 드래그 축 제한.

## 3. Period Band 레인 배치

기간 노드가 시간적으로 겹칠 때, 레인(lane)으로 수직 분리한다.

규칙:
- 짧은 기간이 축에 가장 가까운 레인(lane 0)에 배치된다.
- 긴 기간일수록 위쪽 레인에 배치된다.
- 겹침 판정은 시간 범위 기준. 같은 레인 내에서 시간이 겹치면 다음 레인으로 밀린다.
- 레인 할당은 greedy 알고리즘: 짧은 것부터 정렬 → 가장 아래 빈 레인에 배치.

```
  lane 2  ┌──────────────────────────────────────┐   (가장 긴 기간)
  lane 1     ┌────────────┐   ┌───────────────┐
  lane 0  ┌─────┐  ┌─────┐                          (가장 짧은 기간)
  ─────────────────────────────────────────────────  axis
```

## 4. 축 구성 요소

### 축 라인
전체 캔버스 너비를 가로지르는 수평선. 캔버스 좌표계의 고정 Y 위치.

### 눈금 (Ticks)
일정 간격으로 축 위아래에 짧은 수직선. 간격은 layout_config의 tick_interval로 설정.

### 눈금 라벨
각 눈금 아래에 값 + 단위 접미사. layout_config의 unit 설정 사용.
예: tick_interval=100, unit="년" → "100년", "200년", "300년", ...

### 커넥터 (Connectors)
각 occurrence 노드에서 축까지 이어지는 dashed 수직선. 노드의 시간 위치에서 축까지 직선.

### 마커 (Markers)
축 위에 각 occurrence 노드의 시간 위치에 표시되는 작은 원. span 노드(기간)에는 마커가 없다 — 밴드 자체가 표현.

## 5. 배치 알고리즘

### 메타데이터에서 시간값 추출

플러그인은 ConceptWorkspace가 변환해준 표준 메타데이터만 읽는다:
- `time_value` (필수): 시간 위치를 나타내는 숫자.
- `end_time_value` (선택): 기간 노드의 종료 시간.
- `role`: "period" 또는 "occurrence".

### X 좌표 계산

모든 노드의 X 좌표는 시간값 × TIME_SCALE(상수, 예: 200px/단위)로 결정된다.

기간 노드는 start와 end의 중간점이 X 좌표가 되고, width = (end - start) × TIME_SCALE.

### Y 좌표 계산

기간 노드: 축 위. 레인 배치 알고리즘으로 Y가 결정된다. 사용자 조절 불가.

사건 노드: 축 아래 기본 오프셋. 사용자가 저장한 Y값이 있으면 그 값을 사용 (레인 조절용).

### 충돌 회피 (Occurrence)

사건 노드끼리 X 범위가 겹칠 때, 아래로 밀어내서 스태킹한다.
- 노드를 X 기준 정렬.
- 앞선 노드와 X 범위가 겹치고 Y가 같은 높이에 있으면, Y를 아래로 밀어 카드 높이 + 갭만큼 이동.
- 사용자가 수동으로 Y를 조절한 경우 충돌 회피를 적용하지 않는다 (저장된 Y 우선).

## 6. 인터랙션

### 팬
X, Y 모두 자유. 기본적으로 수평 스크롤이 주 사용이지만, 노드가 많아서 수직으로 넓어질 수 있으므로 Y도 허용.

### 줌
기존과 동일한 zoom-toward-cursor. 줌 변경 시 시간 스케일이 함께 변하므로 눈금 밀도가 달라지는 효과.

### 노드 드래그 (Occurrence)

**X축 드래그 = 시간 변경.** 드래그 종료 시 새 X 좌표를 시간값으로 역산하여 concept_properties를 업데이트한다. `new_time_value = new_x / TIME_SCALE`.

**Y축 드래그 = 레인 이동.** 순수 시각적 배치. position_y만 업데이트.

둘 다 자유롭게 허용하되, 드롭 후 X → 시간값 역산이 핵심.

### 노드 드래그 (Period Band)

밴드 전체를 X축 방향으로 드래그 → start, end 모두 같은 양만큼 이동 → concept_properties의 time_value, end_time_value 모두 업데이트.

### Span Resize (Period Band)

밴드 양쪽 끝에 리사이즈 핸들. 드래그하면 start 또는 end 시간값만 변경된다.

- 왼쪽 핸들 드래그 → time_value 변경 (시작 시간)
- 오른쪽 핸들 드래그 → end_time_value 변경 (종료 시간)
- 드래그 중 실시간으로 밴드 너비가 변한다 (시각적 피드백).
- 드래그 종료 시 역산된 시간값을 concept_properties에 저장.

### 노드 생성

캔버스 우클릭 시 생성 메뉴. 생성된 노드는 클릭한 X 좌표에서 시간값을 역산하여 초기 time_value를 설정한다. Y 좌표로 축 위/아래를 판단하여 적절한 기본 역할(period/occurrence)을 제안할 수 있다.

## 7. 실시간 시각 피드백

드래그 중에는 DB에 값을 저장하지 않고, 오프셋만으로 시각적 위치를 업데이트한다.

- 노드 드래그 중: 카드 위치 + 커넥터 + 마커가 모두 오프셋에 따라 이동.
- 밴드 드래그 중: 밴드 전체가 이동하고, 커넥터도 따라 이동.
- Span resize 중: 밴드의 한쪽 끝만 이동하고 너비가 변한다.

마우스를 놓는 순간에만 최종 좌표를 역산하여 저장한다.

## 8. layout_config 구조

```
{
  // 필드 매핑: archetype별로 플러그인 요구 필드 ← 사용자 도메인 필드
  field_mappings: {
    [archetype_id]: {
      time_value: field_id,       // 필수
      end_time_value: field_id,   // 선택 (있으면 기간 노드 가능)
      role: "period" | "occurrence"  // 이 archetype의 기본 역할
    }
  },

  // 축 표시 설정
  unit: "년",              // 눈금 라벨 접미사
  tick_interval: 1,        // 눈금 간격 (시간값 단위)
  direction: "asc"         // "asc" (좌→우 증가) 또는 "desc" (좌→우 감소)
}
```

매핑이 없는 archetype의 노드가 타임라인 캔버스에 추가되면, 매핑 설정을 유도하는 UI를 보여준다.

## 9. none-type Concept 처리

archetype이 없는(none-type) concept은 필드 매핑이 불가능하다. properties 자체가 없으므로 시간값을 가질 수 없다.

처리 방안:
- 타임라인 캔버스에 none-type concept 추가 시, archetype 지정을 요구하거나 경고를 표시한다.
- 또는 매핑 없이 "위치 고정 노드"로 취급 — 시간축과 무관하게 사용자가 놓은 자리에 그대로 둔다. 커넥터와 마커는 생성하지 않는다. 메모, 주석 용도.

## 10. 레이아웃 전환 시 동작

### freeform → timeline 전환

기존 position_x/y는 보존(freeform 복귀용). 타임라인 플러그인이 concept_properties의 시간값으로 새로운 배치를 계산한다. 시간값 매핑이 안 된 노드는 축 바깥에 "미배치" 상태로 표시.

### timeline → freeform 전환

저장된 position_x/y가 있으면 복원. 없으면(타임라인에서만 사용했으면) 타임라인에서의 마지막 위치를 position_x/y로 저장한 뒤 전환.
