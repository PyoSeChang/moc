## Breaking Changes
- Canvas 중심 모델/에디터 흐름을 Network 중심으로 전환
- Concept/File 중심 캔버스 노드 흐름을 Object 기반 network node 흐름으로 확장
- archetype_ref 필드의 Concept 선택 동작을 대상 Archetype의 properties 중첩 편집 동작으로 변경

## Features
- Network, Context, Object 탐색/편집 UI 추가
- Concept, Archetype, RelationType, Context, Network를 같은 편집 셸 패턴으로 정리
- Object panel, object picker, network object browser를 추가하여 concept/file/context/type 객체를 탐색/배치하는 흐름 추가
- Archetype 목록과 RelationType 목록에 폴더/그룹 기반 정리 흐름 추가
- 필드 타입 선택 난이도 설정과 권장 타입 필터링 추가
- select, multi-select, radio 필드에서 정적 option과 Archetype의 Concept option을 함께 사용할 수 있게 확장

## Fixes
- TypeSelector, Select, ArchetypeRefPicker 드롭다운을 viewport 기준으로 anchor 아래에 재계산하여 editor scroll 중 위치가 고정되는 문제 완화
- embedded terminal의 Ctrl/Cmd+C, Ctrl/Cmd+V 키 처리 보강

## Internal
- Archetype option 설정 직렬화 helper와 field complexity test 추가
- 관계형 Archetype use flow와 UI 재구성 계획 문서 추가
- master의 desktop-app@0.8.0 변경과 network/object UI branch를 병합

## Cross-Package Impact
- renderer는 shared의 Network/Object/Archetype field 타입과 core repository의 network/object/context/type group 흐름을 전제로 동작함
- 전체 테스트는 better-sqlite3 native module을 현재 Node ABI로 rebuild한 뒤 다시 실행해야 함
