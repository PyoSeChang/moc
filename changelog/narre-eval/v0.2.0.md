## Features
- 시나리오를 단일 YAML에서 폴더 기반 구조로 전환 (scenario.yaml + seed.ts + responder.ts + fixtures/)
- 시나리오 타입 구분 추가 (single-turn / conversation)
- 이름 있는 verify 형식 도입 — 모든 체크 항목에 의도를 명시하는 name 필드 필수
- tool assertion 추가 — 특정 도구 호출 횟수 검증 가능
- SeedContext 도입 — seed.ts에서 DB + 파일시스템 시드를 스크립트로 관리
- ResponderContext 도입 — conversation 타입에서 UI 카드(propose/ask/confirm) 자동 응답
- 시나리오별 results/ 저장 — 전역 results 디렉토리 제거

## Internal
- ScenarioResult 구조 통합 (dbAssertions + responseAssertions → verifyResults)
- harness에 tempDir 생성/삭제 추가
- loader에서 seed.ts/responder.ts dynamic import
- runner에서 SSE card 이벤트 감지 → responder → POST /chat/respond 파이프라인
