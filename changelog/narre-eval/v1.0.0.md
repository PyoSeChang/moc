## Breaking Changes
- `scenario.yaml` 단일 파일 형식 폐기, `manifest.yaml` 기반 번들 구조로 전환
- `runner.ts` 제거, `EvalAgentAdapter` + `session-runner` 구조로 교체
- `ScenarioResult` 스키마 변경: `runId`, `agent`, `status`, `metrics`, `scenarioVersion`, `schemaVersion`, `gradingVersion`, `comparison`, `skipReason` 추가
- `Transcript`에 `cardResponseCount`, `sessionResumeCount` 필드 추가
- `TurnTranscript`에 `errors[]` 필드 추가

## Features
- 멀티턴 세션 연속성: narre-server SSE `done` 이벤트의 `sessionId`를 확보하여 conversation 시나리오에서 실제 세션 유지
- 시나리오 번들 구조: `manifest.yaml`, `turns.yaml`, `verify/*.yaml`, `rubrics/*.yaml` 분리
- `EvalAgentAdapter` 인터페이스 + `NarreServerAdapter` 구현 (서버 lifecycle, SSE 파싱, 인라인 카드 응답)
- Universal metrics: turn_count, tool_call_count, unique_tools_used, latency_ms, error_count, card_response_count, session_resume_count (토큰 메트릭은 unsupported로 기록)
- 실행 결과 `runs/` 디렉토리 구조: `run.json` + 시나리오별 `result.json` + `transcript.json`
- 강화된 검증: `db_row_match`, `side_effect`, `tool.sequence`, `tool_absent_in_turn`
- 매니페스트 버전 메타데이터 (`scenario_version`, `schema_version`) 결과에 전달
- `grading_version` / `judge_version` 상수로 채점 계약 추적
- 실행 호환성 검사: `supported_agents`, `required_capabilities`로 비호환 시나리오 `skipped` 처리
- 베이스라인 비교: `--baseline` CLI 옵션, 이전 실행 대비 상태/검증/메트릭 변화 추적

## Internal
- PID 파일 기반 ownership-safe 프로세스 관리 (절대로 자기가 띄우지 않은 프로세스를 kill하지 않음)
- 시나리오별 고유 DB 경로 (repeat 격리)
- SSE 파서 통합 (sendChat/sendChatWithResponder 중복 제거)
- 카드 응답 실패 시 soft error로 기록 (errors[] + no_error 검증으로 감지)

## Cross-Package Impact
- `narre-server`의 SSE `done` 이벤트에 `sessionId` 필드가 필요함 (별도 커밋으로 반영 완료)
- `@netior/shared`의 `NarreStreamEvent`에 `sessionId?: string` 추가 (별도 커밋으로 반영 완료)
